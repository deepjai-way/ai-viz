# AI 可视化方法论

> 版本：1.0  
> 定位：通用核心（工具无关）

---

## 一、核心原则

### 1. 知识源优先

图表不是目标，知识的可视化呈现才是。源是设计文档、代码、DDL，图是视图。不创造新格式，不维护额外的"架构描述文件"——设计文档本身就是最好的源。

### 2. 图种是同一知识的不同投影面

同一份架构知识可以渲染为架构图（结构视角）、时序图（行为视角）、部署图（运维视角）。选择哪种图，取决于要回答什么问题。

### 3. AI 是翻译器，不是画板

AI 的价值不是"你告诉它每个节点怎么摆"，而是从非结构化知识中提取结构、选择最合适的视觉表达、按规格渲染。

### 4. 格式无关性

无论输出 Excalidraw / Draw.io / Mermaid，方法论层面不区分。格式只是最后一步的选择，取决于受众、使用场景、是否需要版本管理。

## 三、模式区分

| 模式 | 触发条件 | 行为 |
|------|---------|------|
| Create | 首次生成，无已有图 | 从知识源全量生成 |
| Edit | 基于已有图修改 | 定向增量编辑，保留已有布局 |

- **Create 模式**：AI 从知识源提取结构，选择图种和格式，按渲染规格全量生成。
- **Edit 模式**：AI 定位修改目标，在保留已有布局的前提下做最小变更。

## 四、设计语言注入

方法论是全局的，但每个项目有自己的视觉风格。通过项目级的 `design-language.yaml` 配置注入。

### 4.1 配置文件格式

```yaml
# design-language.yaml — 项目设计语言配置
project: "项目名称"

colors:
  primary: "#a5d8ff"       # 主要元素（核心服务/节点）
  secondary: "#b2f2bb"     # 次要元素（外部系统/辅助）
  accent: "#ffd43b"        # 强调（网关/关键路径）
  muted: "#e9ecef"         # 弱化（基础设施/背景）
  danger: "#ffc9c9"        # 告警/错误
  text: "#1e1e1e"          # 文字/边框

typography:
  font_family: "default"
  font_size_title: 24
  font_size_body: 16
  font_size_caption: 12

layout:
  direction: "top-to-bottom"
  grid_size: 10
  h_gap: 250
  v_gap: 180
  margin: 50

format_preference:
  internal: "excalidraw"       # 内部文档默认格式
  external: "drawio"           # 对外文档默认格式
  documentation: "mermaid"     # README/文档默认格式

style_preset: "handdrawn"
```

### 4.2 新项目接入

1. 复制 `design-language.yaml` 模板到项目
2. 填入项目配色和偏好
3. 开始使用——AI 编程助手读取配置，按项目风格渲染

## 五、渲染规格体系（specs/）

### 5.1 组织原则

- 按**知识领域**组织，不按输出格式
- **随用随写**，不预建空框架
- 每份 spec 是持久的设计规范，不是一次性提示词

### 5.2 目录结构

```
specs/
├── architecture/          # 架构类
│   ├── layered.md         # 分层架构
│   ├── microservice.md    # 微服务拓扑
│   └── domain.md          # 领域边界
├── behavior/              # 行为类
│   ├── sequence.md        # 时序图
│   ├── flowchart.md       # 流程图
│   └── state.md           # 状态图
├── data/                  # 数据类
│   ├── er.md              # ER 图
│   └── dataflow.md        # 数据流图
├── concept/               # 概念/思维类
│   ├── timeline.md        # 阶段演进图
│   ├── comparison.md      # 对比矩阵图
│   └── convergence.md     # 辐射汇聚图
└── style-presets/         # 风格预设
    ├── academic-bw.md     # 学术黑白
    ├── corporate.md       # 商务风
    └── handdrawn.md       # 手绘风
```

### 5.3 单份 spec 必含字段

```markdown
# {图种名称} 渲染规格

## 适用场景
何时使用此图种，什么知识源适合。

## 布局规范
- 方向（自上而下 / 自左向右）
- 画布建议尺寸
- 元素间距
- 复杂度分级规则

## 配色规范
- 使用 design-language.yaml 中的色彩槽位（primary / secondary / accent / muted）
- 语义映射（如：服务=primary，数据库=muted，网关=accent）

## 元素规范
- 节点形状选择
- 连线样式（实线/虚线/箭头类型）
- 标注要求（连线必须标明传递内容）

## 最小完整示例
一个骨架级示例结构（不含具体业务内容）。
```

## 六、术语表

| 术语 | 含义 |
|------|------|
| 知识源 | 图表信息的来源（设计文档/代码/DDL 等） |
| 图种 | 图表的类型（架构图/时序图/流程图等） |
| 渲染规格 (spec) | 特定图种的设计规范（布局/配色/元素/标注） |
| 设计语言 | 项目级的视觉风格定义（配色/字体/间距），由 `design-language.yaml` 承载 |
| 风格预设 (preset) | 预定义的视觉风格模板（学术/商务/手绘） |
| Create 模式 | 从知识源全量生成新图 |
| Edit 模式 | 基于已有图做定向增量修改 |
