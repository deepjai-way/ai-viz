# 质量控制

> 图表生成与迭代的质量保障体系

---

## 一、通用自检清单

生成任何图表后，AI 编程助手应逐项检查：

### 结构完整性

- [ ] 所有知识源中提到的核心实体均已在图中体现
- [ ] 实体间关系完整，无遗漏的关键连接
- [ ] 图表类型与知识源匹配（未错选图种）

### 布局质量

- [ ] 无元素重叠
- [ ] 布局方向一致（不混合左右和上下）
- [ ] 元素间距均匀，留白充分
- [ ] 分层/分组逻辑清晰

### 标注规范

- [ ] 所有连线有标注（说明传递内容或关系类型）
- [ ] 标签文字未截断，完整可读
- [ ] 节点命名一致（同一实体在全图使用相同名称）

### 视觉规范

- [ ] 配色符合 `design-language.yaml` 定义
- [ ] 元素数量合理（单图不超过 20 个核心节点）
- [ ] 字体大小层次分明（标题 > 正文 > 注释）

## 二、格式特定检查要点

### Excalidraw

| 检查项 | 说明 |
|--------|------|
| JSON 合法性 | 文件可被 JSON 解析器正确解析 |
| ID 唯一性 | 所有元素 id 不重复 |
| 字体设置 | fontFamily 与 design-language.yaml 一致 |
| 绑定关系 | 箭头的 startBinding/endBinding 指向正确元素 |
| 分组完整 | groupIds 关联正确，无孤立分组引用 |
| 坐标合理 | 元素不超出合理画布范围，无负坐标溢出 |

### Draw.io

| 检查项 | 说明 |
|--------|------|
| XML 标签闭合 | 文档结构完整，标签正确嵌套 |
| mxCell ID 唯一 | 所有单元格 id 不重复 |
| 特殊字符转义 | &、<、>、" 等正确转义 |
| 连线端点 | source/target 引用的 id 存在 |
| 样式完整 | style 属性包含必要的视觉参数 |
| 父子关系 | parent 属性指向正确的容器 |

### Mermaid

| 检查项 | 说明 |
|--------|------|
| 语法正确 | 可被 Mermaid 渲染器解析 |
| 节点 ID 合法 | 不含特殊字符导致解析失败 |
| 方向声明 | graph/flowchart 方向与内容匹配 |
| 子图嵌套 | subgraph 正确闭合 |
| 标签转义 | 含特殊字符的标签用引号包裹 |

## 三、迭代协议

### 3.1 基本原则

- 用户反馈 → 定向编辑（Edit 模式），不全量重生成
- 每次编辑保留已有布局，只修改涉及的部分
- 保持修改可追溯（告知用户改了什么）

### 3.2 迭代约束

| 规则 | 说明 |
|------|------|
| 最小变更原则 | 只修改用户指出的问题，不擅自调整其他部分 |
| 布局稳定性 | 未涉及的节点保持原有位置和样式 |
| 迭代上限 | 同一图表最多 5 轮迭代，超过建议拆分或重新梳理需求 |
| 变更确认 | 每轮迭代后说明修改内容 |

### 3.3 何时重新生成

以下情况应放弃 Edit 模式，重新从 Create 模式开始：

- 需求变更超过 50%（大部分节点需要调整）
- 图种需要变更（如从流程图改为时序图）
- 布局结构根本性变化（如从 3 层改为 5 层）
- 迭代超过 5 轮仍未收敛

## 四、美学原则

### 4.1 分层清晰

- 水平分 3-4 层，每层一个色系
- 层间用间距或分隔线明确区分
- 信息流向与层次方向一致

### 4.2 颜色克制

- 主色不超过 3 种（primary + accent + muted）
- 同一语义在全图使用相同颜色
- 高对比度保证可读性（特别是文字与背景）

### 4.3 箭头标注

- 每条连接线标明传递内容
- 箭头方向与数据/控制流方向一致
- 双向关系使用双向箭头或两条单向线

### 4.4 留白充分

- 避免信息过载，元素间保持足够间距
- 重要元素周围留白更多，形成视觉焦点
- 图表边缘预留边距

### 4.5 复杂度管理

| 复杂度 | 节点数 | 策略 |
|--------|--------|------|
| 简单 | ≤ 8 | 紧凑布局，单区域 |
| 中等 | 9-15 | 分组布局，2-3 个逻辑区域 |
| 复杂 | 16-20 | 分层布局，扩展路由走廊 |
| 超复杂 | > 20 | 拆分为多张图，每张聚焦一个子系统 |

### 4.6 一致性

- 同类型元素使用相同形状和大小
- 同一图中字体样式统一
- 间距和对齐网格化
