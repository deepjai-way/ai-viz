# 图种路由

> 从知识源到可视化输出的决策指南

## 一、知识源 → 图种映射

| 知识源类型 | 适用图种 | 推荐插件 | 回答的问题 |
|-----------|---------|---------|----------|
| 系统/服务设计文档 | 系统架构图 | drawio / excalidraw | 系统长什么样？ |
| 接口规范 / 代码调用链 | 时序图 | mermaid / drawio | 请求怎么流转？ |
| 业务规则 / 需求文档 | 流程图 | mermaid / drawio | 业务怎么走？ |
| 代码（类/接口） | 类图 | mermaid / drawio | 对象怎么组织？ |
| 数据库 DDL / 实体设计 | ER 图 | mermaid / drawio | 数据怎么存？ |
| 需求文档 / 用户故事 | 用例图 | drawio / excalidraw | 系统给谁用？ |
| 运维配置 / K8s YAML | 部署图 | drawio / excalidraw | 怎么部署？ |
| 业务状态机 | 状态图 | mermaid / drawio | 状态怎么变？ |
| 架构决策 / 数据互通设计 | 数据流图 | drawio / excalidraw | 数据从哪来到哪去？ |
| 演进/历史叙述 | 阶段演进图（Timeline） | excalidraw / drawio | 事物怎么发展的？ |
| 并列对比分析 | 对比矩阵图（Comparison） | excalidraw / drawio | 几个东西有什么区别？ |
| 因果/条件汇聚 | 辐射汇聚图（Convergence） | excalidraw / drawio | 什么导致了这个结果？ |
| 分类体系/概念层级 | 概念树（Taxonomy） | drawio / mermaid | 这个领域怎么分类？ |
| 概念关系描述 | 概念关系图（Concept Map） | excalidraw / drawio | 概念之间什么关系？ |
| 科普文章/博客帖子 | 文章配图 | ian-illustrator | 如何让抽象概念直观化？ |
| 概念解释/认知隐喻 | 概念图解 | ian-illustrator | 如何可视化一个心智模型？ |
| 方法论/教程内容 | 正文配图 | ian-illustrator | 如何为教学内容搭配视觉？ |
| 通用自然语言描述（非技术架构） | 文章配图 | ian-illustrator | 如何为文字增添视觉吸引力？ |

## 二、置信度分级策略

AI 编程助手根据场景确定性程度，采取不同行为：

| 场景 | AI 行为 |
|------|--------|
| 用户明确指定图种（"画个时序图"） | 直接执行 |
| 知识源与图种强映射（DDL → ER 图） | 直接执行，附说明理由 |
| 知识源可映射多种图种 | 推荐首选 + 列出备选，等确认 |
| 高成本格式（需要额外工具链支持） | 推荐 + 等确认 |
| 对外正式文档 | 推荐 + 等确认 |

### 决策原则

- **确定性高** → 直接执行，减少交互摩擦
- **确定性中** → 给出推荐理由，附带备选项
- **确定性低** → 询问确认，避免浪费生成成本

## 三、主动触发信号

当 AI 编程助手在对话中识别到以下信号时，应主动建议可视化：

| 信号 | 说明 |
|------|------|
| 描述涉及 ≥ 3 个交互组件 | 系统结构复杂度超出文字描述效率 |
| 出现分层/分级结构描述 | 天然适合架构图 |
| 出现时序性描述（先…再…然后…） | 适合时序图或流程图 |
| 出现条件分支（如果…否则…） | 适合流程图或状态图 |
| 用户正在写设计文档且描述复杂架构 | 图文配合提升文档质量 |
| 出现对比/并列描述 | 适合对比矩阵图 |
| 出现因果链条 | 适合辐射汇聚图 |
| 内容面向非技术读者（科普/解释类） | 适合文章配图（ian-illustrator） |
| 需要“画出一个认知动作/隐喻” | 适合概念图解（ian-illustrator） |
| 用户提到“配图”、“文章插画”、“手绘风” | 适合 ian-illustrator |

### 建议模板

```
检测到您正在描述 [信号类型]，建议生成 [图种] 来可视化呈现。
推荐格式：[格式]（理由：[简述]）
是否需要我生成？
```

## 四、格式选择指南

### 4.1 格式特性对比

| 特性 | Excalidraw | Draw.io | Mermaid | ian-illustrator |
|------|-----------|---------|---------|-----------------|
| 视觉风格 | 手绘风 | 专业正式 | 简洁文本 | 艺术手绘风（IP 角色） |
| 文件格式 | JSON | XML | 代码块 | PNG（生成图片） |
| 版本控制友好度 | 中 | 低 | 高 | 低（二进制） |
| 编辑便捷性 | VS Code 插件可编辑 | 专用编辑器 | 纯文本编辑 | 通过 prompt 重新生成 |
| 导出能力 | PNG/SVG | PNG/SVG/PDF | 平台渲染 | PNG 直接输出 |
| 复杂度上限 | 中高 | 高 | 中 | 单概念/单场景 |

### 4.2 场景推荐

| 场景 | 推荐格式 | 理由 |
|------|---------|------|
| 内部设计文档 | Excalidraw | 手绘风、轻量可编辑 |
| 对外客户方案 | Draw.io → PNG/SVG | 专业正式、高清导出 |
| README / Git 文档 | Mermaid | 纯文本、版本可控、平台渲染 |
| PPT / 汇报 | Draw.io → PNG | 高清导出、专业配色 |
| 快速白板讨论 | Excalidraw | 最快出图、手绘感亲切 |
| 论文 / 学术 | Draw.io（学术黑白风格预设） | 符合出版规范 |
| 科普/博客文章 | ian-illustrator | 艺术配图、认知隐喻可视化 |
| 概念解释/教程 | ian-illustrator | 以角色驱动画面让抽象概念具象化 |

### 4.3 决策流程

```
用户需要图表
  │
  ├─ design-language.yaml 中定义了 format_preference？
  │     ├─ 是 → 按配置选择格式
  │     └─ 否 → 继续判断
  │
  ├─ 内容类型？
  │     ├─ 技术工程内容（架构/流程/数据） → drawio / excalidraw / mermaid
  │     └─ 科普/通用/概念解释类内容 → ian-illustrator
  │
  ├─ 内部使用？ → Excalidraw
  ├─ 对外正式？ → Draw.io
  └─ 文档内嵌？ → Mermaid
```

## 五、模式区分

| 模式 | 触发条件 | 行为 |
|------|---------|------|
| Create | 首次生成，无已有图 | 从知识源全量生成 |
| Edit | 基于已有图修改 | 定向增量编辑，保留已有布局 |

### Create 模式流程

1. 分析知识源 → 确定图种
2. 加载对应 spec + `design-language.yaml`
3. 全量生成图表
4. 自检 → 输出

### Edit 模式流程

1. 读取已有图文件
2. 定位修改目标
3. 增量编辑（保留布局、样式、未涉及节点）
4. 自检 → 输出
