# Excalidraw 手绘风格图表生成指南

基于 AI 可视化方法论，从知识源（设计文档/代码/自然语言）提取结构，生成 Excalidraw 格式图表。

---

## 一、触发与路由

### 1.1 显式触发

用户明确说"画架构图"/"画时序图"/"画流程图"/"生成 excalidraw"时直接执行。

### 1.2 主动触发信号

在对话中识别到以下信号时，主动建议可视化：
- 描述涉及 >= 3 个交互组件
- 出现分层/分级结构
- 出现时序性描述（先…再…然后…）
- 出现条件分支（如果…否则…）

主动建议格式："这个架构比较复杂，需要我生成一张架构图来辅助理解吗？"

### 1.3 图种路由（置信度分级）

| 场景 | 行为 |
|------|------|
| 用户明确指定图种 | 直接执行 |
| 知识源与图种强映射（如接口设计 → 时序图） | 直接执行，附说明 |
| 知识源可映射多种图种 | 推荐首选 + 列出备选，等确认 |

### 1.4 支持的图种

| 图种 | 适用场景 | 知识源 |
|------|---------|--------|
| 分层架构图 | 系统全貌、服务拓扑 | 设计文档 |
| 时序图 | 请求链路、鉴权流程 | 接口规范/代码调用链 |
| 流程图 | 业务流程、审批流 | 需求文档/业务规则 |
| 数据流图 | 数据流转、ETL | 架构决策 |
| 泳道图 | 跨部门协作 | 业务流程 |
| ER 图 | 数据模型 | DDL/实体设计 |

## 二、执行流程

### 2.1 Create 模式（首次生成）

```
Step 1: 知识源读取
  - 如果用户指定了文档路径 → 阅读文档，提取相关章节
  - 如果是自然语言描述 → 直接从描述中提取结构

Step 2: 图种确认
  - 高置信度 → 直接进入 Step 3
  - 低置信度 → 推荐图种 + 备选，等确认

Step 3: 结构提取
  - 识别节点（实体/服务/参与者）
  - 识别关系（调用/依赖/数据流）
  - 识别层次（分层/分组/泳道）

Step 4: 布局规划
  - 加载对应图种的渲染规格
  - 加载设计语言配色（参见 design-language.yaml）
  - 计算坐标网格

Step 5: 生成 JSON + 自检
  - 生成 .excalidraw 文件
  - 执行自检清单
  - 输出文件 + 摘要
```

### 2.2 Edit 模式（修改已有图）

用户基于已有 .excalidraw 文件请求修改时：
- 读取已有文件
- 仅修改涉及的部分（新增/删除/移动节点）
- 保留已有布局和样式
- 不全量重生成

## 三、设计语言

### 3.1 色彩系统（参数化）

优先读取项目配置 `design-language.yaml`。若无配置，使用以下默认值：

| 槽位 | 默认色 | 语义 |
|------|--------|------|
| primary | `#a5d8ff` | 核心业务服务 |
| secondary | `#b2f2bb` | 外部系统/接入层 |
| accent | `#ffd43b` | 网关/关键节点 |
| muted | `#e9ecef` | 基础设施（Nginx/DB/Redis） |
| danger | `#ffc9c9` | 告警/需关注 |
| text | `#1e1e1e` | 文字/边框 |

### 3.2 美学原则（强制）

1. **分层清晰**：水平分 3-4 层，每层一个色系
2. **颜色克制**：主色不超过 3 种（primary + accent + muted）
3. **箭头标注**：每条连接线必须标明传递内容
4. **留白充分**：避免信息过载
5. **复杂度分级**：简单图紧凑，复杂图扩展间距

### 3.3 布局规范

| 上下文 | 间距 |
|--------|------|
| 水平元素间距 | 200-300px |
| 垂直层间距 | 150-200px |
| 边距 | >= 50px |
| 箭头到方框间隙 | 20-30px |
| 网格对齐 | 10px 倍数 |

### 3.4 复杂度分级

| 节点数 | 策略 |
|--------|------|
| <= 6 | 紧凑（h_gap=200, v_gap=150） |
| 7-12 | 标准（h_gap=250, v_gap=180） |
| 13-20 | 宽松（h_gap=300, v_gap=200） |
| > 20 | 建议拆分为多张图 |

## 四、Excalidraw JSON 技术规范

### 4.1 文件结构

```json
{
  "type": "excalidraw",
  "version": 2,
  "source": "https://excalidraw.com",
  "elements": [],
  "appState": {
    "viewBackgroundColor": "#ffffff",
    "gridSize": 20
  },
  "files": {}
}
```

### 4.2 元素类型

| 类型 | 用途 |
|------|------|
| rectangle | 服务节点、模块方框、参与者 |
| ellipse | 数据库、外部实体 |
| diamond | 决策点、条件判断 |
| arrow | 连接箭头 |
| text | 标签文字 |
| line | 生命线（时序图）、分隔线 |

### 4.3 必须属性

```
id: 唯一标识（语义化命名，如 "node-api-gateway"）
type: 元素类型
x, y: 坐标（左上角原点，X右增，Y下增）
width, height: 尺寸
strokeColor: "#1e1e1e"
backgroundColor: 对应色彩槽位
fillStyle: "solid"
strokeWidth: 2
strokeStyle: "solid"（实线）或 "dashed"（虚线）
roughness: 1（手绘感）
opacity: 100
fontFamily: 5（Excalifont，所有文字必须用此字体）
fontSize: 16-24
roundness: { "type": 3 }（圆角矩形）
```

### 4.4 箭头定义

```json
{
  "type": "arrow",
  "x": "起点X",
  "y": "起点Y",
  "width": "|终点X - 起点X|",
  "height": "|终点Y - 起点Y|",
  "points": [[0, 0], ["偏移X", "偏移Y"]],
  "roundness": { "type": 2 },
  "startArrowhead": null,
  "endArrowhead": "arrow"
}
```

### 4.5 文字宽高估算

- 英文宽度 ≈ 字符数 × fontSize × 0.6
- 中文宽度 ≈ 字符数 × fontSize × 1.0
- 高度 ≈ fontSize × 1.25 × 行数

## 五、质量自检

生成后必须检查：

- [ ] 无元素重叠
- [ ] 所有连线有标注（说明传递内容）
- [ ] 标签文字未截断（宽度足够）
- [ ] 布局方向一致（不混合左右和上下）
- [ ] 配色符合设计语言
- [ ] 元素数量合理（单图 <= 20 核心节点）
- [ ] 所有 ID 唯一
- [ ] fontFamily 全部为 5
- [ ] JSON 格式合法

---

## 六、输出规范

每次生成后提供：

1. `.excalidraw` 文件
2. 摘要：图种 + 节点数 + 关键结构说明
3. 打开方式：VS Code Excalidraw 插件 / excalidraw.com
4. 如有问题：说明自检发现的问题及修复情况

---

## 七、迭代协议

- 用户反馈修改需求 → Edit 模式，定向修改
- 保留已有布局，只改变更部分
- 最多 5 轮迭代
- 超过建议拆分或重新梳理需求
