# 分层架构图 渲染规格

## 适用场景

- 需要展示"从上到下 N 层"的系统全貌
- 涉及"接入层 → 网关层 → 服务层 → 数据层"等分层结构
- 设计文档中出现"分层"、"自上而下"、"N 层架构"等描述

## 布局规范

- **方向**：自上而下（top-to-bottom）
- **画布建议尺寸**：宽 900-1200px，高按层数 × 180px 估算
- **层间距**：150-200px
- **同层节点间距**：200-280px
- **底层扇出**：允许同层多节点水平排列（如多个下游微服务）

### 复杂度分级

| 节点总数 | 间距策略 |
|---------|---------|
| <= 6 | 紧凑（h_gap=200, v_gap=150） |
| 7-12 | 标准（h_gap=250, v_gap=180） |
| 13-20 | 宽松（h_gap=300, v_gap=200），增加路由走廊 |
| > 20 | 建议拆分为多张图 |

## 配色规范

使用设计语言色彩槽位：

| 层级类型 | 色彩槽位 | 说明 |
|---------|---------|------|
| 外部/接入层 | secondary | 绿色系，表示外部来源 |
| 网关/核心层 | accent | 黄色系，视觉焦点 |
| 业务服务层 | primary | 蓝色系，核心业务 |
| 基础设施层 | muted | 灰色系，支撑角色 |
| 数据库/存储 | muted | 灰色系 + 圆柱形状 |

## 元素规范

### 节点

| 元素 | 形状 | 尺寸建议 |
|------|------|---------|
| 服务/应用 | 圆角矩形 | 宽 200-350px，高 65-80px |
| 数据库 | 圆柱形（ellipse 或 cylinder） | 宽 160px，高 55px |
| 外部系统 | 圆角矩形（虚线边框） | 同服务 |
| 分层背景 | 大矩形（低透明度填充） | 覆盖整层宽度 |

### 连线

| 关系 | 样式 | 标注要求 |
|------|------|---------|
| 同步调用 | 实线 + 箭头 | 标注协议/传递内容（如 "HTTPS"、"Feign"） |
| 异步消息 | 虚线 + 箭头 | 标注消息类型（如 "MQ"、"Event"） |
| 内部对比链路 | 虚线 + 灰色 | 弱化处理 |

### 标注

- 每个节点包含：主标签（服务名）+ 副标签（一句话职责）
- 每条箭头必须标注传递内容
- 右侧可添加步骤注解（① ② ③ 式）

## 最小完整示例结构

```
Layer 1: [外部实体A]
    │ (协议标注)
Layer 2: [网关/代理B]
    │ (传递内容标注)
Layer 3: [核心服务C]
    │ (调用方式标注)
Layer 4: [服务D] [服务E] [服务F]
```
