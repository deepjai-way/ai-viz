# 时序图 渲染规格

## 适用场景

- 需要展示"请求在多个参与者之间的流转顺序"
- 涉及"先调A、再调B、然后B回调C"等时序性描述
- 设计文档中出现"请求链路"、"调用时序"、"交互流程"等描述
- 接口规范、鉴权流程、消息传递链路

## 布局规范

- **方向**：自上而下（时间轴向下流动）
- **参与者排列**：水平排列在顶部
- **参与者间距**：200-300px
- **消息间距（纵向）**：60-80px
- **画布建议尺寸**：宽按参与者数 × 250px，高按消息数 × 70px

### 复杂度分级

| 参与者数 | 消息数 | 策略 |
|---------|-------|------|
| <= 4 | <= 8 | 紧凑 |
| 5-6 | 9-15 | 标准 |
| > 6 | > 15 | 建议拆分为多张图 |

## 配色规范

| 元素 | 色彩槽位 | 说明 |
|------|---------|------|
| 参与者框 | primary | 主角色用主色 |
| 外部参与者 | secondary | 外部系统 |
| 生命线 | text (50% opacity) | 垂直虚线 |
| 同步消息 | text | 实线箭头 |
| 异步消息 | text | 虚线箭头 |
| 返回消息 | muted | 虚线灰色箭头 |
| 激活框 | primary (20% opacity) | 矩形条，表示处理中 |

## 元素规范

### 参与者

| 元素 | 形状 | 说明 |
|------|------|------|
| 参与者头 | 圆角矩形 | 顶部，标注角色名 |
| 生命线 | 垂直虚线 | 从参与者向下延伸 |
| 激活框 | 窄矩形（宽 16px） | 表示该参与者正在处理 |

### 消息（水平箭头）

| 消息类型 | 箭头样式 | 线型 |
|---------|---------|------|
| 同步请求 | 实心三角箭头 | 实线 |
| 异步请求 | 开放箭头 | 实线 |
| 返回/响应 | 开放箭头 | 虚线 |
| 自调用 | 弯曲回到自身 | 实线 |

### 标注

- 每条消息线上方标注：方法名/动作 + 关键参数
- 格式：`动作(关键参数)` 或 `协议: 动作描述`
- 示例：`POST /openapi/token`、`验证 accessToken`、`返回 LoginVO`

## 最小完整示例结构

```
[Client]          [Gateway]         [Service]
    │                 │                  │
    │── 请求A ──────→│                  │
    │                 │── 转发/处理 ───→│
    │                 │                  │
    │                 │←── 响应 ────────│
    │←── 最终响应 ───│                  │
```

## 参考模板

- 适用预设：handdrawn（内部）、corporate（对外）
- 在 excalidraw 中用矩形 + 竖线 + 水平箭头组合实现
- 在 mermaid 中用 `sequenceDiagram` 语法
