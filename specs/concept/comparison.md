# 对比矩阵图（Comparison）渲染规格

## 适用场景

- 知识源包含"A vs B vs C"的并列对比分析
- 需要回答"几个东西有什么区别？各有什么特点？"
- 典型知识源：技术选型对比、方案评估、概念辨析、产品对比

## 布局规范

- **方向**：多列并排（left-to-right），每列一个对比对象
- **画布建议尺寸**：宽度按列数 × 400px 估算，高度按行数 × 100px + 200px
- **列间距（h_gap）**：60-100px
- **行间距（v_gap）**：15-20px（行间紧凑以增强视觉对比效果）
- **维度标签列**：左侧留出 100-140px 宽度放维度名称

### 复杂度分级

| 对比对象数 × 维度数 | 策略 |
|-------------------|------|
| 2×3 ~ 3×4 | 紧凑（col_width=280, row_height=70） |
| 3×5 ~ 4×5 | 标准（col_width=320, row_height=80） |
| > 4×6 | 建议拆分或精简维度，避免信息过载 |

### 结构组成

```
[标题]

[维度标签列] | [对象1列] | [对象2列] | [对象3列]
─────────────────────────────────────────────────
 维度1         内容1-1     内容2-1     内容3-1
 维度2         内容1-2     内容2-2     内容3-2
 ...
─────────────────────────────────────────────────
              [────── 底部总结条 ──────]
```

## 配色规范

### 核心原则

- **每列一色**，通过颜色快速区分不同对比对象
- 列色彩之间应有足够对比度
- 行内格子统一白底（#ffffff），通过列标题色建立归属感

### 配色方案

| 列位置 | 色彩槽位 | 用途 |
|--------|---------|------|
| 列 1 标题 | primary | 第一个对比对象 |
| 列 2 标题 | secondary | 第二个对比对象 |
| 列 3 标题 | danger（浅红）或 accent | 第三个对比对象 |
| 列 4 标题（如有） | accent | 第四个对比对象 |
| 行格子 | 白底 + 浅灰边框 | 内容区域，不抢视觉焦点 |
| 维度标签 | 无背景，muted 文字色 | 维度名称，辅助定位 |
| 底部总结条 | muted 背景 | 总结对比结论 |

### 语义变体

当对比涉及"推荐/不推荐"时：
- 推荐列标题用 secondary（绿=正面）
- 不推荐列标题用 danger（红=警示）
- 中性列用 primary（蓝=客观）

## 元素规范

### 列标题（Header）

| 属性 | 规范 |
|------|------|
| 形状 | 圆角矩形 |
| 高度 | 60-80px |
| 背景 | 对应色彩槽位 |
| 字号 | 22-24px |
| 对齐 | 水平居中 |

### 行格子（Cell）

| 属性 | 规范 |
|------|------|
| 形状 | 圆角矩形 |
| 高度 | 按内容自适应（最小 60px） |
| 背景 | #ffffff |
| 边框 | 浅灰 #dee2e6，1px |
| 字号 | 16px |
| 对齐 | 水平居中、垂直居中 |

### 维度标签

| 属性 | 规范 |
|------|------|
| 位置 | 左侧，垂直居中对齐对应行 |
| 字号 | 16px |
| 颜色 | #888888 |
| 对齐 | 右对齐（靠近格子） |

### 列分隔线

| 属性 | 规范 |
|------|------|
| 样式 | 竖向虚线 |
| 颜色 | #adb5bd |
| 粗细 | 1px |
| 范围 | 从标题下方到最后一行下方 |

### 底部总结条

| 属性 | 规范 |
|------|------|
| 形状 | 圆角矩形 |
| 背景 | muted（#e9ecef） |
| 宽度 | 横跨所有列 |
| 高度 | 50px |
| 字号 | 18-20px |
| 内容 | 一句话对比结论（如"规则执行 vs 规律学习 vs 内容生成"） |

## 最小完整示例结构

```
[标题: A vs B vs C]

           [对象A]    [对象B]    [对象C]
            蓝底        绿底       红底
维度1      [内容]     [内容]     [内容]
维度2      [内容]     [内容]     [内容]
维度3      [内容]     [内容]     [内容]

[────── 总结：核心区别一句话 ──────]
```

## 质量自检补充项

- [ ] 所有列宽度一致（视觉对齐）
- [ ] 所有同行格子高度一致
- [ ] 维度标签与对应行垂直居中对齐
- [ ] 列标题颜色有足够区分度
- [ ] 格子内文字未溢出边界
- [ ] 同一维度的对比内容粒度一致（不要一个写一句话另一个写一段话）

## 实战案例

- 《传统软件 vs AI vs 生成式AI》：3 列 × 5 维度，底部总结条，蓝/绿/红三色标题
